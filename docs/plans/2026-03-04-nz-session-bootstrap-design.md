# Дизайн: одноразовий ручний вхід і збереження сесії nz.ua

## Мета
Додати в додаток режим одноразової ручної авторизації, який зберігає browser session state у локальний файл, щоб щоденний збір міг працювати без повторного введення логіну/паролю.

## Погоджений підхід
Реалізувати окрему CLI-команду `bootstrap-session`:
1. відкриває браузер (не headless),
2. переходить на `nz.ua`,
3. натискає кнопку входу,
4. чекає, поки користувач вручну завершить авторизацію,
5. зберігає `storage_state` у `config/nz_session_state.json`.

## Архітектурні зміни
- `config.py`: нове поле конфігурації `session_state_path`.
- `cli.py`: нова команда `bootstrap-session`.
- `session_bootstrap.py`: окремий модуль ручного входу та збереження сесії.
- `collector.py`: повторне використання збереженої сесії перед fallback-логіном по паролю.

## Безпека
- Секрети залишаються в `.env`.
- Файл сесії (`config/nz_session_state.json`) додається в `.gitignore`.
- Логи не містять пароль.

## Очікуваний UX
1. Користувач запускає `bootstrap-session`.
2. Додаток відкриває вікно браузера і друкує інструкцію.
3. Після ручного входу користувач натискає Enter у терміналі.
4. Сесія зберігається; `run-daily` використовує її автоматично.

## Відоме обмеження
Cloudflare може блокувати автоматичний `POST /login` у headless-середовищі. Ручний bootstrap із реальною сесією використовується як робочий обхід.
