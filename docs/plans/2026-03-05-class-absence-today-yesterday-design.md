# Дизайн: файл "відсутність-сьогодні-вчора" і українізація student-absence-summary

## 1. Мета
Додати новий щоденний CSV-файл з кількістю відсутніх учнів по класах за сьогодні і вчора, а також українізувати заголовки у `student-absence-summary.csv`.

## 2. Новий файл
- Назва: `відсутність-сьогодні-вчора.csv`
- Заголовки: `клас,к-сть сьогодні,к-сть вчора`
- Перший рядок: `усього,<сума сьогодні>,<сума вчора>`
- Далі: рядки по класах.

## 3. Правила розрахунку
### 3.1 Визначення "кількості відсутніх"
- Беремо **унікальних учнів** у класі, які мають хоча б один `ABSENT` у відповідний день.

### 3.2 Дні
- `сьогодні` = `run_date`
- `вчора` = `run_date - 1 день`

### 3.3 Набір класів
- Включаємо всі класи, що присутні у даних за сьогодні або вчора (навіть якщо `ABSENT=0`).

### 3.4 Сортування класів
- За спаданням номера класу: `11`, `10`, `9`, ...
- В межах однакового номера: за літерою по зростанню (`а`, `б`, `в`, ...)
- Приклад: `11-а, 11-б, 11-в, 10-а, ...`

## 4. Зміни у student-absence-summary.csv
Заголовки переводяться на українську:
- `ID учня`
- `Учень`
- `Клас`
- `Н (7 днів)`
- `Н (30 днів)`
- `Н`
- `К-сть періодів 10+`
- `Останній період 10+ (від-до)`

## 5. Зміни в архітектурі
- `analytics.py`: функція обчислення агрегації по класах за сьогодні/вчора + сортування класів.
- `pipeline.py`: виклик нової аналітики і передача результату в reporting.
- `reporting.py`: запис `відсутність-сьогодні-вчора.csv` та українізація заголовків `student-absence-summary.csv`.

## 6. Тестування
- Unit: підрахунок унікальних відсутніх учнів по класах за сьогодні/вчора і порядок класів.
- Integration: наявність нового файлу, коректний рядок `усього`, коректні заголовки.
- Regression: повний прогін `unittest`.
