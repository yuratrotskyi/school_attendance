# Базова інструкція запуску звітності

## 1. Підготовка перед першим запуском
1. Призначте відповідального за ранковий звіт.
2. Переконайтесь, що доступ до `nz.ua` активний.
3. Підготуйте робочу таблицю зведення (Google Sheets/Excel).
4. Узгодьте канал доставки звіту директору.
5. Перевірте актуальність правил у `attendance-policy.md`.

## 2. Щоденний запуск (операційний цикл)
1. `06:00-06:20`: зберіть дані по відвідуваності з `nz.ua` за періоди 7/30/семестр.
2. Оновіть зведення KPI і таблиці по класах.
3. Перевірте інциденти "втік" за правилом `1+`.
4. Сформуйте PDF-звіт і додайте деталізацію за потреби.
5. Надішліть звіт адміністрації до `06:30`.

## 3. Перевірка інцидентів
- До `10:00` класні керівники підтверджують або закривають нові інциденти.
- До `12:00` у зведенні мають бути фінальні статуси.

## 4. Fallback при неповних даних
Якщо на момент ранкового запуску дані в `nz.ua` неповні:
1. Позначте звіт як "попередній".
2. Надішліть попередню версію до `06:30`.
3. Після уточнення даних надішліть оновлену версію до `09:00`.

## 5. Чек перед відправкою
- Перевірено дати і періоди (7/30/семестр).
- KPI перераховані без помилок.
- Інциденти "втік" не містять очевидних хибних спрацювань.
- Вказано статуси інцидентів і відповідальних.
- Файл звіту підписано датою і часом формування.
