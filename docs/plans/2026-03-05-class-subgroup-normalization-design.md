# Дизайн: об'єднання підгруп у назві класу

## Мета
Нормалізувати назви класів так, щоб підгрупи типу `8-В(І підгрупа)` / `8-В (ІІ підгрупа)` / `8-В (профіль ...)` зводилися до одного класу `8-В`.

## Правило нормалізації
- Прибрати будь-який суфікс у круглих дужках після назви класу.
- Нормалізувати пробіли.
- Приклади:
  - `8-В(І підгрупа)` -> `8-В`
  - `8-В (ІІ підгрупа)` -> `8-В`
  - `8-В (профіль математика)` -> `8-В`

## Де застосовується
- У `collector.py` під час нормалізації журналних рядків (до запису raw csv).
- У `parser.py` під час читання csv (backward compatibility для старих файлів).

## Архітектура
- Спільний helper `normalize_class_name(...)` в окремому модулі.
- `collector` і `parser` імпортують helper і використовують одне правило.

## Наслідки
- Всі звіти та агрегації автоматично працюють по об'єднаному класу.
- Дедуплікація і сортування стабілізуються для підгруп.

## Тести
- Unit helper: різні формати дужок.
- Unit parser: `class_name` без підгрупи.
- Unit collector: клас із заголовка/рядка після нормалізації.
